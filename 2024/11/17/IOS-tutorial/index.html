import SwiftUI

struct ContentView: View {
    var body: some View {
        Text("Hello world")
    }
}
  • View 是一个协议（类似于Java中的接口）
    • 所有SwiftUI中的界面元素都必须遵循这个协议。任何遵循View协议的结构体或者类都要定义一个返回某个视图内容的body属性。
  • struct是Swift中的一种数据结构，表示值类型，而class表示引用类型。
    • struct和class的区别在于，struct的实例是按值传递的，而class的实例是按引用传递的。
  • 什么是协议
    • View协议的定义是非常简洁的。它只有一个要求，那就是实现一个只读的body属性
    • 这里的body是计算属性，它的定义就是大括号括起来
1
2
3
4
protocol View {
    associatedtype Body: View
    @ViewBuilder var body: Self.Body { get }
}

Modifier

  • Modifier 是一个用于修改视图属性的方法
    • 它不会改变原始视图，而是返回一个新的视图。你实际上是创建了一个新的视图，而不是修改原始视图的状态。
    • Modifier的链式调用： 修饰符是链式的，这意味着你可以连续使用多个修饰符来配置同一个视图。

Padding

让组件四周留出一定的空白，提升整体界面的视觉美感。

HStack/VStack/ZStack

  • HStack 是一个水平布局容器，可以将子视图在水平方向上排列成一行。
  • VStack 是一个垂直布局容器，适用于将子视图在垂直方向上排列成一列。
  • ZStack 是一个层叠布局容器，适用于在相同坐标上将子视图按顺序叠放。它的布局方式类似于“图层”。

Trailing Closure syntax

animate 函数接收一个闭包作为参数，通常写法如下：

1
2
3
animate(duration: 2.0, animations: {
    print("Animation block")
})

如果闭包是函数的最后一个参数，可以将闭包放到括号外面，并省略参数标签，使代码更简洁：

1
2
3
animate(duration: 2.0) {
    print("Animation block")
}
